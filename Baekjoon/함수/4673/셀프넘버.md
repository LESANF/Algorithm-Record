### 셀프 넘버

## 문제

```
셀프 넘버는 1949년 인도 수학자 D.R. Kaprekar가 이름 붙였다. 양의 정수 n에 대해서 d(n)을 n과 n의 각 자리수를 더하는 함수라고 정의하자. 예를 들어, d(75) = 75+7+5 = 87이다.

양의 정수 n이 주어졌을 때, 이 수를 시작해서 n, d(n), d(d(n)), d(d(d(n))), ...과 같은 무한 수열을 만들 수 있다.

예를 들어, 33으로 시작한다면 다음 수는 33 + 3 + 3 = 39이고, 그 다음 수는 39 + 3 + 9 = 51, 다음 수는 51 + 5 + 1 = 57이다. 이런식으로 다음과 같은 수열을 만들 수 있다.

33, 39, 51, 57, 69, 84, 96, 111, 114, 120, 123, 129, 141, ...

n을 d(n)의 생성자라고 한다. 위의 수열에서 33은 39의 생성자이고, 39는 51의 생성자, 51은 57의 생성자이다. 생성자가 한 개보다 많은 경우도 있다. 예를 들어, 101은 생성자가 2개(91과 100) 있다.

생성자가 없는 숫자를 셀프 넘버라고 한다. 100보다 작은 셀프 넘버는 총 13개가 있다. 1, 3, 5, 7, 9, 20, 31, 42, 53, 64, 75, 86, 97

10000보다 작거나 같은 셀프 넘버를 한 줄에 하나씩 출력하는 프로그램을 작성하시오.
```

## 입력

```
입력은 없다.
```

```
10,000보다 작거나 같은 셀프 넘버를 한 줄에 하나씩 증가하는 순서로 출력한다.
```

## 내 풀이

```js
셀프넘버를 구하는 문제다.

d(n)은 n + n의 각자리수이다. ex) 3은 3 + 3, 33은 33 + 3 + 3

33으로 만들어지는 숫자는 39이다. 33은 39의 생성자이다.

해당 식으로 39를 만들려면 33이라는 숫자가 필요하다.

만약 만들어진다면 33은 생성자이다. 누구의 ? 39의.

어떤 수에 대하여 그 수가 만들어지기 위한 수가 없으면 셀프넘버이다.

7은 셀프넘버이다. 7을 만들 수 있는 숫자가 있나 ? 없다. 7은 생성자가 존재하지 않는다. 고로 셀프넘버이다.

생성자가 아닌 숫자가 아니라, 생성자를 가지지 못한 수를 구하는 문제이다.

범위는 10000보다 작다고 가정한다.

0은 식에 넣어도 만들 수 있는 숫자가 없으므로, 제외한다.

셀프넘버를 가지고 식에 대입하면 어떠한 숫자가나옴.

* 셀프넘버는 생성자를 가지지 못할 뿐이지. 본인은 생성자가 된다.

0은 생성자가 될 수도 없고, 생성자를 가지지도 못하기 때문에 셀프가 아니다.

7은 식에 대입하면 14라는 숫자가 나온다. 14의 생성자는 7이다.

7은 생성자가 됐지만, 7을 만들어주는 숫자가 없다. 즉 생성자를 가지지 못하는 숫자이다.

0은 제외하고 1부터 10000까지 범위의 숫자를 d(n)의 식에 대입하여 나온 결과를 하나의 배열에 넣어준다.

(1). 우린 어떠한 숫자로 식을 대입하여 생성자를 가지는 숫자들의 배열을 구했다.

(2). 생성자를 가지는 배열을 구했다는 것은, 셀프넘버가 아닌 숫자들의 집합을 구했다는 것이다.

(3). 1부터 10000까지 숫자중 셀프넘버가 아닌 숫자들의 집합에서 존재하지 않는 수가 셀프넘버이다.
```

## 내 제출

```js
const dnAry = [];

const dn = (n) => {
    let sum = n;
    while (n != 0) {
        sum += n % 10;
        n = parseInt(n / 10);
    }
    return sum;
};

for (let i = 1; i <= 10000; i++) {
    dnAry.push(dn(i));
}

for (let j = 1; j <= 10000; j++) {
    if (dnAry.indexOf(j) === -1) console.log(j);
}
```
